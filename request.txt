Here is the list of actions and their descriptions:
Action: write-file
Description: This action writes to a file.
Example:
{
    "action": "write-file",
    "args": [
        "file-name",
        "file-content"
    ],
    "description": "This action writes to a file."
}

Action: delete-file
Description: This action deletes a file.
Example:
{
    "action": "delete-file",
    "args": [
        "file-name"
    ],
    "description": "This action deletes a file."
}

Action: list-files
Description: This action lists the files.
Example:
{
    "action": "list-files",
    "description": "This action lists the files."
}

Action: done
Description: This action indicates that work is finished.
Example:
{
    "action": "done",
    "description": "This action indicates that work is finished."
}

Action: read-file
Description: This action read a file.
Example:
{
    "action": "read-file",
    "args": [
        "file-name"
    ],
    "description": "This action read a file."
}



Action should start with <action-start> and end with <action-end> so that it can be parsed.

<task-start>
create a README.md file that summarizes each file
<task-end>
